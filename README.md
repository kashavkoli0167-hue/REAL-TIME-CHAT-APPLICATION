**PROJECT OVERVIEW**
- Real Time Chat Application is a full-stack web application developed to enable instant communication between multiple users.
- The application allows users to send and receive messages in real time without refreshing the page.
- The main objective of this project is to understand real-time communication and client-server interaction.
- This project is inspired by modern messaging platforms such as WhatsApp and Messenger.
 
  This project focuses on implementing real-world web development concepts such as event-driven programming, WebSocket communication, and responsive user interfaces. It provides a practical understanding of how     real-time applications work behind the scenes.

**PROJECT ARCHITECTURE**
- The project follows a client-server architecture.
- It is divided into two main parts: frontend and backend.
- The frontend is developed using React.js with Vite.
- The backend is developed using Node.js and Express.js.
- Socket.IO is used to enable real-time, bi-directional communication.

  This separation of frontend and backend helps keep the codebase clean, modular, and easy to maintain. Each module can be developed and tested independently, which follows industry-standard development practices.

**FRONTEND DESCRIPTION**
- The frontend is located inside the chat-client folder.
- React.js is used to build a component-based and interactive user interface.
- The src folder contains React components, styles, and assets.
- The public folder manages static files such as HTML and icons.
- State management and UI updates are handled efficiently using React.

  The frontend provides a smooth and user-friendly experience. Messages appear instantly on the screen when received, making the application feel fast and responsive.

**BACKEND DESCRIPTION**
- The backend is located inside the chat-server folder.
- Node.js is used as the runtime environment.
- Express.js is used to create and manage the server.
- Socket.IO handles real-time communication between clients.
- The server listens for message events and broadcasts them to connected users.

  The backend is lightweight and scalable. It can be extended easily to include additional APIs, authentication, and database integration in the future.

**WORKING OF THE APPLICATION**
- A user types a message in the chat interface.
- The message is sent to the server using Socket.IO events.
- The server receives the message instantly.
- The server broadcasts the message to all connected clients.
- All users receive the message in real time without page reload.

  This event-based system ensures low latency and smooth real-time communication.

**KEY FEATURES**
